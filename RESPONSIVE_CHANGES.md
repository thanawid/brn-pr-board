# Responsive update

## สิ่งที่ปรับ

- เพิ่ม `responsive.css` เป็นชั้นควบคุม layout ขั้นสุดท้าย เพื่อลดปัญหา CSS เดิมทับกันหลายรอบ
- เพิ่ม `responsive.js` ให้การแตะวันที่บนมือถือเลื่อนไปยังแผงรายละเอียดอัตโนมัติ
- ปฏิทินมือถือแสดงครบ 7 วันโดยไม่ทำให้หน้าเว็บล้นแนวนอน และใช้จุดสีแทนรายการของแต่ละวัน
- ปรับ Header, Hero, การ์ด, แบบฟอร์ม, บอร์ดงาน, Prompt และ Modal สำหรับมือถือ แท็บเล็ต และเดสก์ท็อป
- เพิ่ม Safe Area, `100dvh`, ขนาดจุดสัมผัส และขนาด input 16px เพื่อรองรับ iPhone/iPad และคีย์บอร์ดบนจอ
- ปิด WebGL บนจอเล็กหรือโหมดประหยัดข้อมูล และซ่อน Canvas อัตโนมัติเมื่อสร้าง WebGL context ไม่สำเร็จ
- อัปเดต Service Worker เป็น cache `brn-pr-v5-responsive`

## การนำขึ้น GitHub Pages

แทนที่ไฟล์ทั้งหมดใน repository เดิม แล้ว Commit/Push ตามปกติ จากนั้นเปิดเว็บใหม่และรีเฟรชหนึ่งครั้ง หากเบราว์เซอร์ยังแสดงของเก่า ให้ปิดแท็บแล้วเปิดใหม่เพื่อให้ Service Worker รุ่นใหม่เข้าควบคุม
