# การตั้งค่าผู้ใช้ — BRN PR Workspace

## สถานะของชุดนี้

- เชื่อม Firebase Authentication โปรเจกต์ `brn-pr-board`
- เข้าสู่ระบบด้วย Google
- จำสถานะการเข้าสู่ระบบบนอุปกรณ์
- มีปุ่มออกจากระบบและแสดงชื่อ/อีเมลผู้ใช้
- ยังไม่ได้ย้ายข้อมูลปฏิทินไป Cloud Firestore ข้อมูลเดิมยังอยู่ในเบราว์เซอร์แต่ละเครื่อง

## โหมดเริ่มต้น

ไฟล์ `auth-config.js` ตั้งค่าเป็น:

```js
accessMode: "google-only"
```

หมายความว่า บัญชี Google ทุกบัญชีสามารถเข้าสู่ Workspace ได้ เหมาะสำหรับทดสอบระบบก่อนจำกัดสมาชิก

## จำกัดเฉพาะผู้ใช้ที่อนุญาต

เปิดไฟล์ `auth-config.js` แล้วแก้เป็น:

```js
accessMode: "allowlist",

allowedEmails: [
  "อีเมลผู้ดูแล@gmail.com",
  "อีเมลเจ้าหน้าที่@gmail.com",
],

adminEmails: [
  "อีเมลผู้ดูแล@gmail.com",
],
```

จากนั้น Commit และ Push ขึ้น GitHub อีกครั้ง

- `allowedEmails` คือรายชื่อผู้ที่เข้า Workspace ได้
- `adminEmails` คือรายชื่อที่แสดงสถานะ “ผู้ดูแลระบบ” ในหน้าเว็บ
- อีเมลต้องตรงกับบัญชี Google และระบบไม่สนตัวพิมพ์เล็ก/ใหญ่

## ข้อจำกัดของขั้นนี้

รายชื่อผู้ใช้ยังอยู่ในไฟล์หน้าเว็บ จึงเหมาะสำหรับการควบคุมการใช้งานเบื้องต้น แต่ยังไม่ใช่ระบบสิทธิ์ฝั่งฐานข้อมูลเต็มรูปแบบ เมื่อย้ายข้อมูลไป Firestore ควรเก็บสมาชิกและบทบาทในฐานข้อมูล พร้อมตั้ง Firestore Security Rules
